## 3. Retrieve `SuperMaterialThemeData` from `BuildContext`

Dart does not support method overloading based only on return types. Therefore, implement two methods with different names.

### Nullable lookup

```dart
static SuperMaterialThemeData? maybeOf(BuildContext context) {
  final theme = Theme.of(context);

  if (theme is SuperMaterialThemeData) {
    return theme;
  }

  return null;
}
```

### Non-null lookup with fallback

```dart
static SuperMaterialThemeData of(BuildContext context) {
  final theme = Theme.of(context);

  if (theme is SuperMaterialThemeData) {
    return theme;
  }

  return theme.brightness == Brightness.dark
      ? SuperMaterialThemeData.dark()
      : SuperMaterialThemeData.light();
}
```

Where appropriate, the fallback theme should derive relevant values from the current `ThemeData` instead of discarding existing application-level theme configuration.

Document the behavior of both methods:

* `maybeOf` returns `null` when the current theme is not a `SuperMaterialThemeData`.
* `of` always returns a valid `SuperMaterialThemeData`.
