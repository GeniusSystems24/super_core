# SuperCore Package — Version 2.0.0 Update Requirements

Update the Flutter package located at `flutter/super_core` to version `2.0.0`.

## Static Variables

* Remove all static values from the `SuperTokens` classes and make them dynamic. These values must be added to `SuperMaterialThemeData` and used across the following packages and their examples:

| Package                             | Theme Class                   |
| ----------------------------------- | ----------------------------- |
| `flutter/super_tab_bar`             | `SuperTabBarTheme`            |
| `flutter/super_auto_suggestion_box` | `SuperAutoSuggestionBoxTheme` |

| `flutter/super_map`                 | `SuperMapTheme`               |

| `flutter/super_navigation_sidebar`  | `SuperNavigationSidebarTheme` |
| `flutter/super_table_field`         | `SuperTableFieldTheme`        |
| `flutter/super_tree`                | `SuperTreeTheme`              |

## Font Family

* Add support for changing the font family by introducing a `fontFamily` property in `SuperMaterialThemeData`.
* Allow passing a `TextTheme` that contains the primary font family so it can be extracted or used based on the value of `mergeTextTheme`.
* When `mergeTextTheme` is set to `true`, use the font defined in the provided `TextTheme` while preserving the default typography settings defined by `SuperMaterialThemeData.textTheme`.

## Widgets

### SuperAppBar

* Copy the Flutter `AppBar` implementation into `SuperAppBar`, while allowing all of its properties to be customized, including height, color, typography, icons, actions, leading widget, title, and all other supported properties.
* Add support for controlling the subtitle position using a `subtitlePosition` property with the following values:
  * `SubtitlePosition.above`
  * `SubtitlePosition.below`
* Add a mechanism for controlling the maximum number of actions displayed in the app bar through a `maxActions` property.
* When the number of actions exceeds the configured limit, display a three-dot overflow button containing the remaining actions.
* By default, determine the action limit responsively based on screen size:
  * Display up to 3 actions on small screens.
  * Display up to 4 actions on medium screens.
  * Display up to 5 actions on large screens.
* Allow the action limit for each screen category to be customized using:
  * `maxMobileActions`
  * `maxTabletActions`
  * `maxDesktopActions`
* Create a `SuperAppBarTheme` class that extends `AppBarTheme` and adds the following properties:
  * `subtitlePosition`
  * `maxActions`
  * `maxMobileActions`
  * `maxTabletActions`
  * `maxDesktopActions`
* Integrate `SuperAppBarTheme` with `SuperMaterialThemeData` so it can define the default configuration for `SuperAppBar`.

### SuperSliverAppBar

* Copy the Flutter `SliverAppBar` implementation into `SuperSliverAppBar`, while allowing all of its properties to be customized, including height, color, typography, icons, actions, leading widget, title, and all other supported properties.

* Add a mechanism for controlling the maximum number of actions displayed in the app bar through a `maxActions` property.
* When the number of actions exceeds the configured limit, display a three-dot overflow button containing the remaining actions.
* By default, determine the action limit responsively based on screen size:

* Use `SuperAppBarTheme` to determine the default configuration for `SuperSliverAppBar`.

### SuperCard

* Add expandable-card support so the card can reveal additional content when the user presses a button or taps the card itself.
* Support both vertical and horizontal expansion directions.
* Add `leading` and `trailing` properties so icons, text, or other widgets can be placed at the beginning and end of the card.
* Create a `SuperCardTheme` class that extends `CardThemeData`.
* Integrate `SuperCardTheme` with `SuperMaterialThemeData` so it can define the default configuration for `SuperCard`.

### SuperDialog

* Remove `SuperDialog`.
